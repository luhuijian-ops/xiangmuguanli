三、用户故事与场景化需求


3.1 核心用户角色及用户故事


角色 典型用户故事
项目负责人 我需要快速创建项目并分配成员，通过甘特图直观查看整体进度，任务延期时能及时收到提醒
开发 / 执行成员 我希望登录后清晰看到自己待办任务，支持上传文件，评论时 @负责人同步进展
团队管理员 我需要管控项目权限，统计成员工时与任务完成率，导出报表给管理层
查看者（客户 / 高管） 我仅需查看项目进度，无需编辑权限，通过日历视图了解关键里程碑

3.2 核心业务场景流程


场景 1：项目创建与初始化

  1. 项目负责人登录系统
  2. 创建项目（名称 / 描述 / 起止时间）
  3. 导入 / 添加成员并分配角色
  4. 创建里程碑
  5. 拆分任务并分配执行人
  6. 设置优先级 / 截止日期

场景 2：任务执行与协作

  1. 成员登录查看「我的任务」
  2. 执行并更新状态（todo→doing）
  3. 上传任务相关文件
  4. 评论区 @负责人同步问题
  5. 完成后标记为 done
  6. 负责人验收确认

场景 3：项目复盘与统计

  1. 管理员进入统计模块
  2. 筛选项目 / 时间范围
  3. 查看完成率 / 工时 / 延期率
  4. 导出报表
  5. 调整后续项目规划

    五、详细交互设计规范


    5.1 通用交互规则


    1. 操作反馈：所有操作给出加载 / 成功 / 错误提示，提示语简洁明确
    2. 弹窗规则：新增 / 编辑用弹窗；删除需二次确认，标红警示
    3. 快捷键支持：
      • Ctrl+N：快速创建任务
      • Esc：关闭弹窗 / 返回
      • Ctrl+S：保存编辑
    4. 空状态处理：无项目 / 无任务 / 无数据时给出引导文案与操作按钮

    5.2 任务看板（Kanban）交互


    1. 支持任务卡片拖拽变更状态
    2. 悬停显示编辑 / 删除 / 查看详情
    3. 颜色区分优先级：P0 红、P1 橙、P2 蓝、P3 灰、P4 浅灰
    4. 支持列宽拖拽、字段显示 / 隐藏
    5. 支持按执行人 / 优先级 / 标签筛选，条件可记忆

    5.3 甘特图交互


    1. 支持任务 / 里程碑拖拽调整时间
    2. 支持日 / 周 / 月粒度切换
    3. 悬停显示任务详情
    4. 依赖任务显示关联线，删除主任务时给出提示

    2. 时间约束：任务开始≤截止；项目结束≥开始；里程碑≤项目结束

    4. 数值约束：工时 0.5-24 小时；优先级 0-4

    • 九、异常处理机制


    9.1 业务异常场景


    异常场景 前端提示 后端返回
    分配给已删除用户 该用户已被删除，请重新选择 code=400，执行人不存在
    文件超大小限制 文件超过 500MB 限制 code=413，大小超限
    项目编码重复 该编码已存在 code=409，重复创建
    任务已过期未完成 任务标红 + 延期提醒 定时扫描 + 日志 + 通知
    微信 / 钉钉授权失败 授权失败，请重试 code=500，第三方授权失败
    并发修改同一任务 任务已被修改，请刷新 乐观锁，code=409

    9.2 系统异常处理


    1. 数据库异常：显示系统暂时不可用，记录日志并告警
    2. 接口超时：加载动画 + 超时提示，后端熔断优化
    3. Token 失效：自动跳登录，提示登录已过期


    十、集成对接细节


    10.1 微信登录（OAuth2.0）


    1. 前端跳转微信授权 URL 获取 code
    2. 后端通过 code 换取 access_token
    3. 获取用户信息并自动注册 / 登录

    十四、测试方案


    14.1 测试类型


    • 功能测试：全流程正向 / 反向用例
    • 接口测试：参数、异常码、权限校验
    • 性能测试：1000 + 并发，接口 < 500ms，页面 < 2s
    • 兼容性测试：Chrome/Edge/Safari/ 移动端浏览器
    • 安全测试：SQL 注入、XSS、CSRF、越权
    • 易用性测试：核心流程≤3 步完成

    14.2 核心测试用例（示例）


    用例 ID 场景 操作 预期
    TC001 创建项目成功 填信息并提交 创建成功，跳转详情
    TC002 项目名称为空 提交空名称 提示不能为空，不提交
    TC003 任务拖拽变更 拖拽 todo→done 状态同步，无刷新
    TC004 微信授权取消 扫码后取消 提示授权取消，返回登录


    十五、运维与监控


    15.1 系统监控


    • 服务器：CPU / 内存 / 磁盘告警阈值
    • 应用：接口量、错误率、响应时间
    • 数据库：慢查询、连接数、主从状态
    • 告警：钉钉 / 邮件 / 短信

    15.2 数据备份


    • 全量备份：每日凌晨 2 点
    • 增量备份：每小时一次
    • 保留策略：全量 30 天，增量 7 天
    • 恢复流程：停机→恢复→校验→启动→验证

    15.3 发布管理


    • 流程：测试→灰度→全量→回滚预案
    • 版本：Git 分支管理，发布打 Tag


    十六、风险评估与应对


    风险类型 风险点 概率 影响 应对
    技术风险 低代码定制不足 中 高 提前 POC，预留原生开发方案
    集成风险 第三方接口变更 低 中 封装适配层，关注平台公告
    性能风险 大数据量卡顿 中 中 索引优化、Redis 缓存、分页
    运维风险 宕机 / 数据故障 低 高 集群、主从、异地备份
    体验风险 操作复杂 中 中 新手引导、提示、可用性测试
